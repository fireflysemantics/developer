Introduction

We will be creating the NgRx demo application from scratch using the Firefly Semantics Slice Lightweight Reactive Application State Manager.

The NgRx Demo allows us to search the Google Books API and place books in our own collection.

We will be doing this incrementally in phases describing the Slice state management implementation details as we go along.

Core Services

Application State

The service src/app/modules/core/services/app-state.service.ts manages the core application state.

The core application state consist of values keyed as follows:

  • APPLICATION_CONFIG: Application configuration. We are not using one for this demo, but include it in general as it's nice to have access to it from the core application state.
  • USER: The authenticated user
  • SIDENAV: Whether the side navigation menu is open or closed.
  • AUTHENTICATION_ERROR: User sign in authentication error.
  • IS_AUTHENTICATED: Whether the user is authenticated.

Each property also has a corresponding Observable reference initialized in the AppStateService constructor:

  • sideNavOpen$: Observable<boolean>

  • isLoggedIn$: Observable<boolean>

  • user$: Observable<User>

  • authenticationError$: Observable<boolean>

    The object store state is contained and managed by the OS:OStore property on the AppStateService. The OS property is initialized using the START configuration like this:

//===================================
//Create the object store instance
//===================================
OS: OStore<ISTART> = new OStore(START);

The START object contains the initial values for the OStore instance. The values are ObsValueReset instances. These instances contain the Observable for the store, the initial value, and the reset value (If it's different from the initial value).

The ISTART interface definition is necessary in order to get IDE autocompletion.

The LOCAL_STORAGE_USER_KEY is used to look up authenticated user instances in local storage.

When the service constructor runs it initializes the user state like this:

const user =
  JSON.parse(localStorage.getItem(LOCAL_STORAGE_USER_KEY)) || null;
this.OS.put(this.OS.S.USER, user);

This way the app will render as authenticated if the user exists.

The method updateUser(user?: User) places the value in local storage on authentication and also initalize the object store property USER.

/**
 * Update user state on sign in and sign out.
 */
updateUser(user?: User) {
  if (user) {
    this.OS.put(this.OS.S.USER, user);
    localStorage.setItem(LOCAL_STORAGE_USER_KEY, JSON.stringify(user));
  } else {
    this.OS.put(this.OS.S.USER, null);
    localStorage.setItem(LOCAL_STORAGE_USER_KEY, null);
  }
}

We derive the value for isLoggedIn$ like this:

//===================================
// Derive the state for IS_AUTHENTICATED
// Based on the user$ observable.
//===================================
this.user$
    .pipe(
      tap(u => {
        this.OS.put(this.OS.S.IS_AUTHENTICATED, !!u);
      })
    )
    .subscribe();
}

The method setAuthenticationError() is used to indicate that an authentication error occurred during signin.

If the LoginComponent calls this method to indicate that an authentication error occurred the UI will process the corresponding observable and display the error.

The method toggleSideNav() is used to open and close the side navigation menu item container.

Security

The src/app/modules/core/services/auth.service.ts manages sign in and sign out.

The login method sets the authentication error if the username does not match test.

If the username matches test then the user state is updated and the application navigates to / which redirects this to /books.

login({ username, password }: Credentials) {
  if (username !== "test") {
    this.s.setAuthenticationError(true);
  } else {
    this.s.updateUser({ name: username });
    this.router.navigate(["/"]);
  }
}

The logout method sets the user state to null and navigate to /login:

logout() {
  this.s.updateUser();
  this.router.navigate(["/login"]);
}

Guard

The src/app/modules/core/services/auth-guard.service.ts secures the route /books.

The canActivate method checks the application state to see if the user is authenticated. If not it routes to /login.

canActivate(): Observable<boolean> {
    const isAuthenticated = this.s.OS.snapshot(this.s.OS.S.IS_AUTHENTICATED);
    if (!isAuthenticated) {
      this.router.navigate(["/login"]);
    }
    return isAuthenticated;
  }
}

Book Services

State

The service src/app/modules/book/services/book-state.service.ts contains methods for updating the user book collection and storing search results.

The bookStore:EStore instance stores books found while conducting a search.

The bookCollection:EStore instance contains books the user adds to their collection.

The books$: Observable<Book[]> is a reference to the Observable of the books contained in the bookStore:EStore.

The collection$: Observable<Book[]> is an Observable of the books contained in the bookCollection:EStore.

The searching$: Observable<boolean> is used to display the spinner when the search API is in progress.

The method onSearch(query:string) is used to set the bookStore.query property to the current query string entered by the user.

This way the bookStore is always aware of what the current query is.

The toggleCollection(book:Book) method is used to add or remove the book argument from the collection.

API

The service src/app/modules/book/services/book-api.service.ts contains the methods used to update book state and query the Google book API.

The search API is triggered through observation of the the bookStore.query within the book-api.service.ts constructor.

This is the code snippet:

s.bookStore
  .observeQuery()
  .pipe(
    filter(Boolean),
    debounceTime(200),
    distinctUntilChanged(),
    tap(async (query: string) => {
      s.bookStore.searching = true;
      const bo: Observable<Book[]> = this.searchAPI(query);
      const books: Book[] = await bo.toPromise();
      s.bookStore.reset();
      s.bookStore.postA(books);
      s.bookStore.searching = false;
    })

When a new search term is observed, we set bookStore.searching to true in order to trigger the spinner. We awaitthe search results and when we receive them we reset the bookStore in order to remove the previous results. We then postA the array of books to the store and set bookStore.searching to false in order to remove the spinner and display the results.

We also pass an error handler to subscribe in order to handle errors:

.subscribe(
  () => {},
  e => {
    s.bookStore.searchError = of(e);
  }
);

Search

The search component src/app/modules/book/components/search.component.ts renders book search results:

<bc-book-preview-list [books]="s.books$ | async"> </bc-book-preview-list>

The query value is captured through the search method which is bound to the input fields keyup event:

(keyup)="search($any($event.target).value)"

The search method binds the query string to the bookStore.query property.

The bookStore.query property is observed ( The setup for this is in the constructor of the book-api.service.ts). and any change to it that produces a defined query string will trigger a search.

The component also has an error$ property which is a reference to the bookStore.searchError observable property.

If an error occurs during search it is bound to the bookStore.searchError property.

The error$ is used by the component to render the error display like this:

<mat-card-footer>
    <span *ngIf="(error$ | async)">
    {{ error$ | async }}
    </span>
</mat-card-footer>

The searching$:Observable<boolean>, which is bound to the AppStateService.searching$ property, displays the spinner like this:

<mat-spinner
  [class.show]="searching$ | async"
  [diameter]="30"
  [strokeWidth]="3">
</mat-spinner>

Book View

The book view component src/app/modules/book/components/book-view.component.ts is a smart component that delegates book detail display to the component src/app/modules/book/components/book-detail.component.ts.

The components properties are initialized like this:

this.book = this.route.snapshot.data["book"];
this.isSelectedBookInCollection$ = this.s.collection$.pipe(
      map(() => this.s.bookCollection.contains(this.book))
);

As can be seen the book property is obtained from the route snapshot via the resolver.

We derive the isSelectedBookInCollection$ state from the AppStateService.collection$ observable such that the any time there is a change to the books in the collection it will update the value.

The bc-book-detail event emitters for add and remove are bound to the method:

toggleCollection() {
    this.s.bookCollection.toggle(this.book);
}
